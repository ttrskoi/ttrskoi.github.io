MStatusManager
系统状态管理接口

欧文航
2026-05

将语言、驾驶限制、日夜主题、STR、关机状态、P 席屏幕可用性等系统状态统一封装，为业务页面提供“可查询、可观察、可回调”的稳定入口。

6+核心系统状态集中管理
2接入模式：主动查询与实时观察
1单例入口，降低业务接入复杂度
MStatusManager Interface Documentation

定位与价值

一个入口收口多类系统状态

  • 1
    降低业务耦合

    业务页面不直接感知底层电源、车辆、配置变化等来源，通过 MStatusManager 获取统一语义。

  • 2
    提升状态一致性

    初始化时注册 STR 电源监听、广播、驾驶限制监控和日夜主题，保证应用内状态同步。

  • 3
    支持实时响应

    既提供轮询查询方法，也提供 MLiveData 和 MStatusCallback，适配不同业务场景。

从“分散读取”到“统一治理”

管理范围

覆盖 UI 运行时最关键的状态面

状态类别代表接口业务意义
用户体验langugeStatus() / themeStatus()语言与日夜主题变化，支撑界面即时切换。
行车安全restrictionStatus()区分操作限制与视频限制，指导页面禁用或内容屏蔽。
电源生命周期getStrStatus() / getShutDownStatus()识别 STR 与关机准备状态，支持页面收敛资源和保存现场。
座舱能力getIsDisplayPSupported() / getVehicleType()判断 P 席屏幕可用性和车型 gbook code，便于能力差异化适配。
状态面覆盖体验、安全、生命周期、硬件能力

接入方式

三种方式适配不同业务节奏

主动查询

  • restrictionStatus()：当前驾驶限制状态
  • langugeStatus()：当前系统语言
  • themeStatus()：当前日夜主题
  • getStrStatus()：当前 STR 状态
  • getShutDownStatus()：当前关机状态

实时订阅

  • getRestrictionData() / getLanguageData() / getThemeData()
  • getVehicleType() 观察车型 gbook code
  • registerStatusCallback() 注册后立即回放当前状态，并继续接收变化
  • unregisterStatusCallback() 页面销毁时解除订阅
查询决定初态，订阅驱动变化

关键状态编码

核心枚举需要在业务侧形成统一认知

驾驶限制四象限

  • 0操作 OFF，视频 OFF
  • 1操作 ON，视频 OFF
  • 2操作 OFF，视频 ON
  • 3操作 ON，视频 ON

生命周期状态

  • STR_OFF = 0，STR_ON = 1
  • SHUTDOWN_PREPARE = 0，SHUTDOWN_NONE = 1
  • THEME_NONE = -1，THEME_DAY = 0，THEME_NIGHT = 1
  • LANGUAGE_EN = 0，LANGUAGE_ZH = 1
状态值简单，但业务含义必须一致

落地流程

应用接入建议：初始化、消费、解绑

Step 01应用启动初始化调用 getInstance() 获取单例，并在启动阶段执行 registerActivity(context)。
Step 02页面读取初始值页面进入时通过查询方法设置语言、主题、驾驶限制等初态。
Step 03订阅实时变化高频交互页面使用 MLiveData 或 MStatusCallback 响应变化。
Step 04销毁时解除注册页面退出时执行 unregisterStatusCallback()，避免无效回调和资源占用。
一次初始化，多页面复用

治理建议

接口稳定之外，还要降低误用成本

  • !
    废弃接口隔离

    gearStatus() 与 getGearData() 已废弃且始终返回 0，建议业务侧不再新增使用。

  • A
    状态语义文档化

    对驾驶限制、STR、关机状态建立统一说明，避免不同页面对同一状态做出不一致反应。

  • T
    测试覆盖关键变化

    重点验证语言切换、日夜主题切换、行车限制变化、STR/关机准备等生命周期场景。

稳定接口 + 使用规范，才能形成框架资产

汇报结论

MStatusManager 是 UIBase 的状态底座

通过统一入口、实时通知和生命周期协同，MStatusManager 将复杂系统状态转化为业务可直接使用的框架能力，适合沉淀为 UIBase 的标准接入规范。
